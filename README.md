# CCT383 App - 产品说明文档

## Demo 页面展示设备
- **目标设备**: iOS Mobile First (iPhone 13 Pro 及以上规格)
- **样式规范**: IOS 样式

## 核心愿景
打造从"灵感发现"到"实地导航"的一站式闭环，通过算法生成线性最优路径，并融入社交化实时指引。

## 目标用户
- 追求个性化体验的自由行者
- 希望高效规划复杂行程的家庭/情侣游客
- 依赖社交媒体种草并需要落地执行的年轻群体

---

## 1. 全局信息架构 (Global Information Architecture)

应用采用底部固定 Tab Bar 结构，包含四个核心模块：

### Tab Bar 模块
1. **Discover (发现)**: 沉浸式活动流与灵感探索
2. **Mapping (规划工作台)**: 地图选点、收藏与智能路径生成的核心工作区
3. **Routes (行程库)**: 个人计划管理与社区精选路线浏览
4. **Community (社群)**: 路线分享、游记动态与旅伴匹配

### 顶部导航
- **全域 Search Bar**: 位于应用顶部，支持全局搜索

---

## 2. 详细页面功能规格 (Page Specifications)

### 2.1 模块一：Discover (发现)

#### 2.1.1 主视图：活动卡片流 (Activity Feed)

**布局规范:**
- 单列垂直滚动
- 无分页，无触底（无限加载）

**卡片组件 (Activity Card):**

**头部信息:**
- 日期标签：醒目展示 (如 "OCT 25" 或 "Today")
- 标题：大字号，清晰易读
- 位置标签：小字号，带图标 (📍)

**视觉主体:**
- 自动滚动图片条：位于标题下方
  - 图片组以匀速自动横向滑动 (Marquee 效果)
  - 用户可手动干预滑动
  - 图片条右侧边缘有渐变遮罩，暗示更多内容

**交互行为:**
- **点击**: 触发 Hero Animation (共享元素转场)
  - 卡片以点击点为中心平滑放大至占据屏幕 90%
  - 背景 Feed 流高斯模糊并变暗
- **关闭**: 在放大态下向下滑动或点击背景遮罩
  - 卡片缩小回原位
  - 背景恢复清晰

#### 2.1.2 详情页动态结构 (Dynamic Detail Modal)

卡片放大后，根据活动类型呈现两种不同的内容流：

**场景 A: 多地点/分布式活动 (Multi-Location Event)**
- **适用**: 节日庆典 (如火把节)、音乐节、城市周
- **内容结构:**
  1. 活动总览：简介、总日期、主视觉
  2. 地点子卡片池 (Vertical Card Pool): 
     - 垂直排列的子卡片，每个代表一个分会场/地点
     - 包含：地点名、具体活动内容、时间
     - 操作：每个子卡片独立支持"加入收藏" (发送至 Mapping 模块)
  3. 滚动：用户在模态层内继续向下滚动查看所有子地点

**场景 B: 单地点/固定展览 (Single-Location Event)**
- **适用**: 博物馆特展、单一景点、餐厅
- **内容结构:**
  1. 详细信息：高清轮播图、富文本介绍、票价/时间、社媒链接、主"加入收藏"按钮
  2. 分区引导符：滚动至底部时，显示居中分割线文本 (如 "----- 猜你喜欢 -----")
  3. 无缝推荐流：紧接分割线后，直接嵌入其他 Activity Cards (缩小版或同款)
     - 逻辑：点击推荐流中的新卡片，当前模态层平滑过渡到新卡片的详情态
     - 目的：杜绝触底 (No Dead End)，保持用户持续探索

---

### 2.2 模块二：Mapping (智能规划工作台)

**核心逻辑:** 融合"探索"与"规划"的双状态工作区。用户在同一页面完成从选点到生成线性路径的全过程。

#### 2.2.1 状态 A：探索模式 (Explore Mode) - 默认状态

**主视图:** 全屏交互式地图

**功能层:**
- **顶部筛选栏:** 
  - 悬浮 Chips (美食/风景/活动/打卡)
  - 控制地图 Pin 的显示
- **地图 Pin:** 不同类别对应不同颜色/图标
- **地点详情抽屉 (Bottom Sheet):** 
  - 点击 Pin 弹出
  - 包含：图片、介绍、评分、社媒链接、**"添加到行程"**按钮

**浮动操作栏:**
- 显示已选点数 (如 🛒 3)
- 当点数 ≥ 2 时，浮现主按钮 "生成路线 (Plan Route)"

#### 2.2.2 状态 B：规划模式 (Planning Mode)

**触发:** 点击"生成路线"，界面平滑过渡。地图背景缩小/模糊，主视区切换为线性水平卡片流。

**核心视图：线性路径流 (Linear Horizontal Path)**
- **布局:** 水平滑动 (scroll-snap)，垂直居中
- **节点卡片 (Node Card):** 代表一个游玩点 (图片、名称、建议时长)
- **通勤连接段 (Commute Segment):** 
  - 嵌入在节点卡片之间
  - 显示交通方式图标、耗时、距离
- **末端概览 (End Cap):** 
  - 最后一个节点后
  - 展示静态地图轨迹预览图 + 行程总计 (总时长/总预算)

**编辑能力:**
- **拖拽排序:** 长按卡片调整顺序，系统自动重算通勤时间
- **删除/替换:** 删除节点，或点击"替换"获取算法推荐的附近同类点
- **约束调整:** 提供预算滑块、交通方式切换开关

**终态操作:**
- 保存行程：存入 Routes 库
- 开始导航：直接进入 Guide 模式
- 返回地图：退回状态 A 继续选点

---

### 2.3 模块三：Routes (行程管理中心)

**核心逻辑:** 个人计划与社区路线的仓库。

**分段控制:** [我的计划] | [社区精选]

**列表项:** 
- 展示路线缩略图 (线性路径简化版)
- 标题、天数、标签、创建者

**核心操作:**
- **编辑:** 加载数据至 Mapping (状态 B) 进行修改
- **开始:** 启动路线，进入 Guide 模式
- **分享:** 生成海报/链接

---

### 2.4 模块四：Community (社群与结伴)

**核心逻辑:** 基于路线的社交网络。

**动态 Feed:** 用户分享的图文路书 (类似小红书)

**旅伴匹配 (Travel Partner Match):**
- **逻辑:** 匹配"相同路线 + 相近时间 + 附近位置"的用户
- **展示:** "User X 计划在周六走'多伦多精华线'，缺 1 人"
- **操作:** 发送 join 请求，开启聊天

---

### 2.5 核心执行引擎：Guide 模式 (Execution Mode)

**核心逻辑:** 全屏覆盖层，将规划路径转化为实时执行指令，融入社交化指引。

**界面布局:** 沿用 Mapping 的水平滑动卡片流样式，保持认知一致性。

**关键组件:**

**通勤指导卡片 (Commute Card):**
- 位于两节点之间
- 内容：动态地图片段、交通大图标、实时路况、**"唤起外部导航"**按钮

**智能 Trip Guide 弹窗 (Smart Pop-up):**
- **触发:** 时间到达 (预设出发前) 或 地理围栏 (到达目的地)
- **内容逻辑 (基于用户偏好):**
  - 场景推荐: 
    - 摄影爱好者 -> 推送拍照姿势/角度参考图
    - 美食家 -> 推送必点菜品测评
  - 行动提醒: "您已停留 45 分钟，建议出发前往下一站 (车程 15min)"
- **交互:** 
  - 倒计时进度条
  - [稍后提醒]、[完成并前往下一站]
- **联动:** 点击"前往下一站"自动滑动至下一个节点卡片并高亮
